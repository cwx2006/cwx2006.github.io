设置界面：

一、Authentication（认证模块）

  • Username（用户名）：用于登录 SheepIt 账号，标识你的渲染节点身份。
  • Password（密码）：对应 SheepIt 账号的密码，用于身份验证。

二、Theme（主题模块）

  • Light/Dark（亮色 / 暗色主题）：可选择界面显示风格，亮色主题以浅色背景为主，暗色主题则采用深色背景，提升不同使用场景下的视觉舒适度。

三、Cache（缓存模块）

  • Working directory（工作目录）：设置 SheepIt 存储临时文件、渲染数据的本地路径，需确保该目录有足够的磁盘空间（建议预留数 GB 以上），多个客户端实例需配置不同工作目录。

四、Compute devices（计算设备模块）

  • CPU/GPU 勾选框

    ：选择用于渲染的硬件设备。

    • CPU：使用中央处理器进行渲染，适合无独立显卡或需要利用闲置 CPU 资源的场景。
    • GPU（如示例中的 NVIDIA P104-100）：使用图形处理器加速渲染，效率通常远高于 CPU（需确保显卡驱动已正确安装，Linux 系统需安装 NVIDIA 专有驱动）。

  • CPU cores（CPU 核心滑块）：调整用于渲染的 CPU 核心数量，可根据系统其他程序的资源需求灵活分配，避免因渲染占用过多核心导致系统卡顿。

  • Memory（内存滑块）：设置 SheepIt 可使用的最大内存容量，需根据项目需求和系统总内存合理配置，防止内存不足导致渲染失败。

  • Priority（优先级滑块）：调节 SheepIt 进程的系统优先级，“Low” 表示低优先级，不影响其他程序运行；“High” 表示高优先级，让渲染任务获得更多系统资源。

五、Advanced options（高级选项模块）

  • Minimize to SysTray（最小化到系统托盘）：启用后，客户端最小化时会隐藏到系统托盘区域，不占用任务栏空间。
  • Block Eevee projects（阻止 Eevee 项目）：若你的硬件对 Blender 的 Eevee 渲染引擎支持不佳，可勾选此选项，避免接收该类型渲染任务。
  • Disable large downloads（禁用大文件下载）：防止因接收大尺寸项目导致带宽或磁盘空间不足。
  • Proxy（代理）：若网络需要通过代理访问，可在此配置代理服务器信息。
  • Computer name（计算机名称）：自定义设备在 SheepIt 农场中的识别名称，便于管理多设备节点。
  • Incompatible process（不兼容进程）：可设置与 SheepIt 冲突的进程名称，客户端会监测并避免在这些进程运行时执行渲染。
  • Max time per frame (in minute)（每帧最大渲染时间）：设置单帧渲染的最长时间限制，超过该时间则终止渲染，防止某一帧因异常占用过多资源。
  • Write settings to file（写入设置到文件）：启用后，配置会保存到本地文件，下次启动时自动加载。
  • Auto sign in（自动登录）：勾选后，客户端启动时会自动使用保存的账号密码登录。
